# Structure

The repository follows a modular flake structure.

```
.
├── config/          # App configs (alacritty, git, nvim, starship)
├── docs/           # Documentation source
├── hosts/         # Machine-specific configs
├── modules/        # Reusable NixOS/Home Manager modules
├── wallpapers/     # Images
├── flake.nix       # Entry point
└── Makefile       # Automation
```

## `config/`

Dotfiles for specific applications.
These are not imported by flake.nix directly; they serve as a reference or can be manually linked/symlinked.
Typically, Home Manager imports the `home-files` or similar mechanisms, but here we often import the `.nix` equivalents in `modules/home`.

- `config/alacritty/`: Alacritty terminal config.
- `config/git/`: Git config.
- `config/nvim/`: Neovim config.
- `config/starship/`: Starship prompt config.

## `hosts/`

Machine-specific NixOS configurations.

- `hosts/default/`: The default template.

Each host directory usually contains:
- `configuration.nix`: The system config.
- `hardware-configuration.nix`: Generated by `nixos-generate-config`.

## `modules/`

The core logic.

- `modules/desktop/`: Desktop environment (display, audio, fonts).
- `modules/home/`: User environment (shell, git, dev, media, etc.).
- `modules/server/`: Server services (ssh).

### Dendritic Pattern

This repo uses a "dendritic" pattern (conceptually similar to [this](https://github.com/divnix/dendritic)) to organize modules.
Instead of one giant file, configuration is split into small, composable files that import each other.

Example: `modules/home/default.nix` imports specific concerns (`shell.nix`, `git.nix`, etc.). `hosts/default/configuration.nix` imports `modules/desktop` and `modules/home`.

This allows you to enable/disable entire subsystems just by commenting out an import line, or by overriding the module in a specific host.

### Example: How Imports Chain

1. `flake.nix` points to `hosts/default/configuration.nix`.
2. `hosts/default/configuration.nix` imports `modules/desktop/default.nix` and `modules/home/default.nix`.
3. `modules/desktop/default.nix` imports `display.nix`, `audio.nix`, `fonts.nix`.
4. `modules/home/default.nix` imports `shell.nix`, `git.nix`, etc.

To change the desktop environment from KDE Plasma to GNOME, you only need to edit `modules/desktop/display.nix` and comment out the KDE block, uncommenting the GNOME block.
